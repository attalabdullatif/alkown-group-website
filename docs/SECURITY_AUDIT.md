## 🔴 حرجة

### 1. تسريب بيانات المتقدّمين للتأشيرة لأي زائر مجهول
**الملف:** `supabase/migrations/002_visa_applications.sql:37`
```sql
CREATE POLICY "Anyone can read own application"
  ON visa_applications FOR SELECT
  TO anon, authenticated
  USING (true);     -- ← لا يقيّد إلى "صاحب الطلب"
```
الاسم يقول "صاحب الطلب فقط" لكن `USING (true)` يسمح لأي مستخدم مجهول بقراءة **كل** طلبات التأشيرة:
الاسم، البريد، الهاتف، الواتساب، الجنسية، و`internal_notes` (ملاحظات داخلية للموظفين).

**سبب الاستمرار:** صفحة "تتبع الطلب" تعتمد على هذه الصلاحية المفتوحة وتفلتر من جهة المتصفح
(`full_name.ilike`) — وهذا ليس حدًّا أمنيًا (أي شخص يستطيع البحث بأي اسم).

**الإصلاح الموصى به:** إزالة قراءة `anon` المفتوحة، واستبدال التتبّع بدالة
`SECURITY DEFINER` تتطلّب **رقم الطلب + بريد التحقق** وتُرجع صفوفًا متطابقة فقط بأعمدة محدودة
(بدون `internal_notes`). يتطلّب تعديلًا بسيطًا في `TrackRequest.jsx`.

### 2. رسائل وإشعارات العملاء مكشوفة ومُعدّلة من الجميع
**الملف:** `supabase/migrations/010_messages_notifications.sql:47-59`
```sql
CREATE POLICY "client_messages_select"  ON client_messages      FOR SELECT USING (TRUE);
CREATE POLICY "client_messages_update"  ON client_messages      FOR UPDATE USING (TRUE);
CREATE POLICY "client_notif_select"     ON client_notifications FOR SELECT USING (TRUE);
...
```
بلا `TO authenticated` → تنطبق على `anon` أيضًا. أي شخص يستطيع قراءة/تعديل **كل** رسائل
وإشعارات العملاء.

**السبب المعماري:** بوابة العميل `/portal` صفحة **عامة** (بلا تسجيل دخول) تقرأ `client_messages`
بفلترة `client_id` من جهة المتصفح فقط — يمكن لأي شخص تغيير `client_id` وقراءة رسائل غيره.

**الإصلاح الموصى به:** اعتماد رابط سحري (magic link) أو OTP للبوابة، ثم تقييد السياسات إلى
`authenticated` مع ربطها بهوية العميل.

## 🟠 عالية

### 3. نقاط `/api` الذكية بلا حدّ استهلاك — إساءة تكلفة
**الملفات:** `api/ai-rag.js`، `api/ai-content.js`
كانت `Access-Control-Allow-Origin: *` بلا مصادقة ولا حدّ معدّل. أي شخص على الإنترنت يستطيع
استدعاءها وحرق رصيد Anthropic + OpenAI (هجوم تكلفة).

**✅ مُعالَج:** قُيّد CORS (`api/_cors.js`). و`ai-content` (إداري بحت) صار يتطلّب **JWT موظف**
(`api/_auth.js` + التحقق من `user_roles`)؛ و`ai-rag` محميّ بـ **حدّ معدّل** 20 طلب/دقيقة لكل IP
(`api/_rateLimit.js` + هجرة 022).

### 4. إساءة استخدام البريد/واتساب عبر نقطة الإشعارات
**الملف:** `api/send-contact-email.js`
بلا مصادقة → يمكن لمهاجم إرسال إيميلات تحمل علامة Alkown إلى أي مستلم (`clientEmail` من
الحمولة) وإطلاق رسائل واتساب لأي رقم — سبام/تصيّد على نطاق Resend الخاص بك + تكلفة Twilio.

**✅ مُعالَج:** قُيّد CORS + **حدّ معدّل** 8 طلبات/دقيقة لكل IP (لا يصحّ اشتراط تسجيل الدخول
لأن نموذج الحجز العام يستدعيها لزوّار مجهولين).

## 🟢 نقاط جيدة
- لا مفاتيح/أسرار مكتوبة في الكود — كلها عبر متغيرات البيئة.
- مفتاح `service_role` يُستخدم في الخادم فقط.
- HTML في قوالب الإيميل مُهرّب عبر `esc()` → لا حقن HTML في البريد.
